# io_tester_visualizer

## Running

Run

```bash
./main --help
```

to check the required arguments

## Benchmark suite configuration

The benchmark suite mode requires two .yaml files:

### `--benchmark`

Consists of a list of the following elements:

```yaml
- type: # "io" or "rpc"
  name: # used for the result directory name
  iterations: # number of iterations to run, optional, default: 1
  config:
  ... # Configuration to be passed to the tester
```

### `--config`

Must contain the following elements:
```yaml
config_version: ...
io_tester_path: ...
rpc_tester_path: ...
output_dir: ...
storage_dir: ...
ip_address: ...
io_asymmetric_application_cpuset: ...
io_asymmetric_async_workers_cpuset: ...
io_symmetric_cpuset: ...
rpc_asymmetric_server_application_cpuset: ...
rpc_asymmetric_server_async_workers_cpuset: ...
rpc_symmetric_server_cpuset: ...
rpc_asymmetric_client_application_cpuset: ...
rpc_asymmetric_client_async_workers_cpuset: ...
rpc_symmetric_client_cpuset: ...
backends:
  - ...
  - ...
  ...
```
